UI:
Check that the room can be booked with valid data
Check that the room can’t be booked with invalid data
Check that the earlier booked dates show as Unavailable

Detailed Test Cases:

Test Case 1: Check that the room can be booked with valid data
Preconditions:
1. User App is open (https://automationintesting.online/)
2. There is at least one available room to book.

Steps:
1. Click the "Book this room" button for an available room.
2. Fill in the "First Name" field with valid data (e.g., "John", length > 3).
3. Fill in the "Last Name" field with valid data (e.g., "Doe", length > 3).
4. Fill in the "Email" field with a valid email address (e.g., "john.doe@email.com").
5. Fill in the "Phone" field with a valid phone number (e.g., "01234567890", length >= 11).
6. Drag to select a valid date range (e.g., 2 consecutive days) on the calendar.
7. Click the "Book" button.

Expected Result:
A modal displaying "Booking Successful!" is shown.


Test Case 2: Check that the room can’t be booked with invalid data
Preconditions:
1. User App is open (https://automationintesting.online/)
2. There is at least one available room to book.

Steps:
1. Click the "Book this room" button for an available room.
2. Fill in the "First Name" field with invalid data (e.g., "A", length < 3).
3. Fill in the "Last Name" field with invalid data (e.g., "B", length < 3).
4. Fill in the "Email" field with an invalid email address (e.g., "invalid-email").
5. Fill in the "Phone" field with an invalid phone number (e.g., "123", length < 11).
6. Drag to select a valid date range on the calendar.
7. Click the "Book" button.

Expected Result:
The booking is not submitted, and error messages are displayed indicating the validation failures (e.g., "size must be between 3 and 18", "must be a well-formed email address", "size must be between 11 and 21").


Test Case 3: Check that the earlier booked dates show as Unavailable
Preconditions:
1. User App is open (https://automationintesting.online/)
2. A specific room has been booked for specific dates (e.g., using Test Case 1 or API).

Steps:
1. Navigate back to the main page or refresh the page.
2. Click the "Book this room" button for the same room booked in the preconditions.
3. Observe the calendar for the booked dates.
4. Attempt to drag and select the exact dates that were previously booked.
5. Attempt to click the "Book" button with valid form data but overlapping dates.

Expected Result:
The previously booked dates are visually rendered as unavailable on the calendar.
The user is unable to create a new booking for the overlapping dates.
